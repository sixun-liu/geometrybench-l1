总体设计：能力金字塔 × 三个核心机制

分级不是分类，低层是高层的前提 —— 可以诊断模型在哪一层断了
L5 工程发现开放 · agent
L4 生成造出合法几何
L3 推理为什么 / 如果
L2 拓扑 ⭐结构怎么连 · 何为合法
L1 识别看得出这是什么
↑ 越往上越开放，需要专家/仿真 ↓ 越往下越封闭，kernel 全自动判分
① Kernel-as-Oracle：kernel 既出题又判卷
拓扑计数、邻接图、布尔结果、合法性诊断都可由 kernel 精确计算 → 题目零人工、无限再生、定期换题防数据污染。Phase 1 的观察"求交是共同主线"在此升级为方法论
② 同源多模态：一题多问，考表示不变性
一个 B-Rep 种子 → STEP / CAD 序列 / mesh / 点云 / 渲染图 / 文本六种表示，同一道题跨表示各问一遍：LLM、VLM、3D 专用模型同台同分制，跨表示一致性本身是指标
③ 分级递进：雷达图 + 等级认证
每级 0–100 分 → 五维雷达图；认证 Level k 要求 L1…Lk 全部 ≥ 60（类比自动驾驶分级）——防止"会生成但不懂拓扑"的模型拿高等级
核心理念：用确定性的几何引擎，度量不确定的学习模型
3/9

L1 识别 · L2 拓扑（本方案最有原创性的层级）

L1 做统一化整合；L2 填补空白 —— "几何像 ≠ 拓扑合法"的质变发生在这里
L1 识别 T1.1 曲面基元识别（平面/圆柱/圆锥/球/环面/样条 + 参数）｜T1.2 加工特征识别（孔/槽/凸台/倒角，MFCAD++ 风格）｜T1.3 语义分类与计数 —— 标签全部来自 kernel 与生成程序，100% 自动判卷
L2 任务示例
T2.1 拓扑计数"几个面？亏格是多少？"（欧拉公式 V−E+F=2−2g 自动验证）
T2.2 邻接问答"面 #3 与面 #7 是否共享边？与圆柱面相邻的面有哪些？"
T2.3 缺陷定位注入缺陷（删面/自交/非流形）→ 判合法性并指出问题位置
T2.4 拓扑等价几何参数不同的两个模型，拓扑是否同构？
为什么 L2 重要：L4 的 validity 瓶颈在这里诊断
HoLa 报告 84% validity → 16% 被 kernel 拒收。生成非法是因为根本不懂拓扑，还是懂但生成时丢失？只看 L4 永远无法回答，L2 单独考拓扑才能回答
样题（六角螺母：六棱柱 + 中心通孔）
[螺母线框渲染图：标注 6 侧面 + 顶/底 + 孔圆柱面]
Q：共几个面？亏格？
A：9 个面、亏格 1
顶/底面各含一个圆形内环——内环不增加面数。区分"真懂 B-Rep 结构"与"数表面块数"的细节
Oracle：kernel TopExp 遍历计数，零人工

L3 推理：模型对几何引擎的"心智模型"有多准

kernel 真实执行反事实分支，答案精确可知 —— 布尔推理直接对接求交研究
任务形式
T3.1 程序→几何给 CAD 序列（不给图）问形状属性；或反向选序列
T3.2 参数反事实"参数从 10 改 20，孔会穿透吗？体积变多少？"
T3.3 布尔推理给 A、B 和布尔操作，预测结果拓扑/连通性
T3.4 对称与分解识别对称面/旋转轴；分解为"基元+布尔"构造树
指标：MCQ 准确率（杜绝评分模糊）+ 反事实一致性（一组只改一个参数的题答案自洽）+ 跨表示一致性
样题 ⭐ 参数反事实（盲孔 → 通孔）
[40×30×20 板
盲孔深 15
8 个面]
→
[孔深改 25
穿透成通孔
7 个面]
Q：孔深 15→25，面数如何变化？A：8→7——孔底平面消失，底面新增内环但面数不变。模型答对的前提是在"心里"执行这次几何运算
样题 · 布尔分裂（连通性变化）
圆柱 A（⌀20×40）− 长方体 B（30×30×10，同心）= ？
A：两个分离实体——长方体从中段切断圆柱。经典的"布尔运算改变连通性"考点，模型预测的正是 kernel 求交算法计算的
5/9

L4 生成：统一三层指标，回应"指标不统一"

任务覆盖 Phase 1 的 2×2 框架全象限；榜单按"直接 B-Rep / CAD 序列"分赛道
任务输入 → 输出对应现有研究
T4.1 条件重建点云 / mesh / 单视图 / 多视图 → B-Rep 或 CAD 序列Point2CAD · ComplexGen · TransCAD · CAD-Recode
T4.2 文本生成自然语言 → CADText2CAD 系
T4.3 编辑补全已有模型 + 指令 → 新模型；残缺序列 → 补全现有评测稀缺
T4.4 无条件生成— → 模型分布BrepGen · HoLa
① 几何保真
Chamfer 距离 · F-score@1mm · 法向一致性
"形状像不像"
② 拓扑合法 — kernel 硬判定
watertight 率 · 自交率 · 非流形率 · 序列可执行率
"kernel 收不收"
③ 工程可用
与参考的拓扑同构率 · 可编辑性 · 参数合理性
"能不能进工程流程"
评分报告样例：法兰盘点云重建（带 4 个螺栓孔）
Chamfer 0.36mm、F1@1mm 0.93（很高）｜watertight ✓｜拓扑同构 ✗：参考 22 面 vs 生成 18 面，螺栓孔只检出 2/4 —— 几何分数很高、工程上不可用，正是 Phase 1 的核心教训：三层缺一不可。只报 Chamfer 奖励"像但非法"，只报 validity 奖励保守的简单形状

L5 工程发现：几何接地的 agent 评测

与 EngiAI 等的本质区别 —— 被测 agent 真正"接触"B-Rep 实体，而非文本问答
T5.1 可制造性审查（DFM）
给 B-Rep 找设计缺陷：薄壁、深孔无法加工、缺工艺圆角、无退刀槽

评判：规则引擎自动判 + 专家金标
指标：检出率 / 误报率
T5.2 目标导向设计优化
"减重 20%，保持孔位与应力上限"——agent 迭代修改 CAD 序列

评判：kernel 验合法 + FEM 验性能
指标：目标达成度（相对专家基线）
T5.3 设计规律发现
给一族参数化设计 + 性能数据，归纳参数-性能规律并外推

评判：held-out 参数组合上验证预测能力（防"编故事"）
指标：外推误差
Agent 评测闭环
Agent（被测） ⇄ CAD Kernel 查询/修改 ⇄ FEM 仿真 / 规则引擎 → 达成度评分
过程指标同时记录：kernel 调用次数、迭代轮数（刻画效率）。仿真离线预计算控制评测成本。规模最小（200–500 例）但含金量最高，是五级中唯一需要实质人工投入的级别。
金字塔顶端：以真实几何为对象的开放式工程能力 —— 现有工程 agent 评测从未做到

路线图 · 预期贡献 · 诚实边界

零期 Pilot 两到三周、一人可完成 —— 我可以立即着手
零期 Pilot（2–3 周）
50 个 ABC 种子 + pythonOCC 跑通最小管线；L2/L3 各 100 题；2–3 个大模型试考。验证：答案唯一性 ≥99%、区分度、成本
一期：L2+L3 全量
空白最大、全自动构造、最体现 kernel 特色，性价比最高
二期：L1+L4
对现有任务/数据/指标做统一化整合
三期：L5
依赖仿真与专家资源，基于前两期反馈设计
预期贡献
  • 首个五级统一的几何智能评价体系（现有全是单级切片）
  • 首个把拓扑理解作为独立考点的 benchmark
  • Kernel-as-Oracle 自动出题/判卷范式，机制性防数据污染
  • 统一 L4 生成指标，解决方法间无法横向比较的问题
  • 几何接地的工程 agent 评测，把"发现"落到真实 B-Rep 上
风险与开放问题（诚实说明）
  • L5 评判客观性：用仿真/规则 oracle + 外推验证压缩主观空间，专家成本仍是瓶颈
  • 复杂 B-Rep 的 STEP 超出 LLM 上下文 → 需紧凑序列化或设复杂度上限
  • "理解" vs "模式匹配"：一致性 + 换题是应对，不构成完备证明
  • DFM/FEM 我尚无实践经验，L5 为框架性设计，盼老师指导细化
GeometryBench：让"几何智能"从一个口号变成一张可以打分的考卷
